# 技术规范

## 技术栈

> 💡 **提示**：遇到不熟悉的技术时，建议先通过 Context7 MCP 查阅相关文档。

- **前端**: HTML5 + CSS + JavaScript
- **后端**: Node.js
- **2D 游戏引擎（可选）**: [Pixi.js](https://pixijs.download/release/pixi.min.js) - 高性能 WebGL 渲染引擎
  - **何时使用**: 需要精灵动画、粒子效果、复杂图形渲染时
  - **何时不用**: 简单的 DOM/Canvas 游戏，原生 API 足够时不必引入
  - **原则**: 按需选择，不为使用而使用
- **2D Canvas（可选）**: 原生 Canvas 2D API
  - **原则**: Pixi.js 与 Canvas 2D 选其一（根据游戏开发场景，选择最合适便捷的），以提升界面效果并降低复杂度

## 环境配置
- **统一环境**: 所有游戏共用根目录的 Node.js 环境，避免重复安装
- **启动方式**: 所有游戏均从项目根目录启动

# AI 集成规范（重点）

## 模型配置
| 用途 | 模型 | 备注 |
|------|------|------|
| LLM 文本生成 | `models/gemini-2.5-flash` | 关闭 thinking 模式 |
| 图片生成 | `imagen-4.0-generate-preview-06-06` | - |

## 配置管理
- 配置文件位置：各游戏根目录下的 `config.json`
- 包含字段：`proxyUrl`、`baseUrl`、`apiKey`
- **禁止**从环境变量读取配置
- 参考实现：`test_gemini_api.js`

## AI 调用要求
1. **日志输出**: 终端/控制台打印每次 LLM 调用的完整日志（方便及时查看状况）
2. **状态管理**: 每个 AI 调用必须有 loading 状态和错误处理
3. **等待优化**: 通过以下方式掩盖 AI 响应等待时间：
   - 过渡/加载动画
   - 符合游戏场景的交互设计
   - 批次预生成内容
4. **视觉反馈**: AI 功能必须通过文字提示/动画/数值变化等形式呈现
5. **提示词质量**: 提示词质量会直接影响 AI 游戏效果，需明确目标、输入、输出格式与约束
6. **上下文管理**: 明确上下文边界与来源，避免无关信息污染结果
7. **错误重试机制**: 提升 AI 调用稳定性，实现自动重试
   - **重试策略**: 使用指数退避（Exponential Backoff），初始延迟 1 秒，每次重试延迟翻倍
   - **最大重试次数**: 建议 3 次，可根据场景调整
   - **可重试错误**: 网络超时、5xx 服务端错误、速率限制（429）、连接重置
   - **不可重试错误**: 4xx 客户端错误（401/403/400 等）、JSON 解析失败（需检查 prompt）
   - **重试日志**: 记录每次重试的原因、次数和延迟时间
   - **降级处理**: 达到最大重试次数后，提供友好的用户提示或使用缓存/默认值

# 项目结构

## 目录规范
- 每个游戏独立一个目录，统一存放于 `games/` 下

## 2D 游戏参考模板

```text
game/
├── assets/          # 资源目录（音频、图片等静态资源）
│   └── music/       # 音效与背景音乐
├── src/             # 源码目录
│   ├── audio/       # 音频系统相关逻辑
│   ├── engine/      # 核心引擎模块（输入、世界、引擎循环等）
│   ├── entities/    # 实体/角色相关
│   ├── game/        # 游戏玩法/场景逻辑
│   ├── platform/    # 平台桥接适配层
│   ├── systems/     # 系统模块
│   ├── ui/          # UI 相关模块
│   └── utils/       # 通用工具
├── index.html
└── README.md
```

# 交互与 UI 设计

## 输入限制
- **允许**: 键盘 + 鼠标
- **禁止**: 语音、摄像头
- 尽量减少纯文字交互（剧情类游戏除外）

## 前端展示要求（复杂/创新玩法游戏）
- ✅ 可视化游戏玩法介绍（界面直接展示或点击查看）
- ✅ 关键信息日志记录（如击败敌人等事件）

## 美术资产
- 音效和美术资产暂不在开发范围
- 可利用图片生成模型创建游戏资产
- 鼓励美化 Web 前端界面

# 开发流程

## 核心原则
1. **TDD 模式**: 对关键逻辑先编写测试再开发
2. **渐进式开发**: 先构建基础游戏循环，再接入 AI 功能

## 代码规范
- 直接输出完整可运行的代码
- 关键逻辑添加**中文注释**
- AI 调用部分使用 `// AI LOGIC` 标注
