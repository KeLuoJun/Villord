# 🏘️ 治村物语 — AI 村长模拟器 策划方案

> **项目代号**：Villord（治村物语）  
> **核心理念**：AI 驱动的村庄经营管理模拟 — 用对话治理村庄，用智慧经营未来  
> **技术栈**：HTML5 + JavaScript（原生）  
> **UI 风格**：平面 2D 面板式管理界面（清爽田园风格）

---

## 一、游戏概述

### 1.1 一句话描述

**"你是一名村长，通过自然语言对话指挥 AI 村民种地、建设、交易，每个村民都有独特性格和自主意志，你需要在有限资源下做出最优决策，带领小村走向繁荣。"**

### 1.2 核心定位

本游戏**不是**一个动作/操作类农场游戏，而是一个**面板式经营管理模拟器**：
- 玩家不亲手种地/浇水，而是以**村长身份**通过对话指挥村民干活
- 没有角色移动、没有可视化地图，所有信息通过**数据面板 + 卡片 + 列表**呈现
- 核心乐趣来自**资源管理、人员调度、AI 对话博弈、经济决策**

### 1.3 核心创新点

| 维度 | 传统经营游戏 | 本游戏创新 |
|-----|------------|----------|
| **操作方式** | 点击拖拽操作 | 自然语言对话指挥村民 |
| **NPC** | 固定脚本自动干活 | AI 村民有个性，可能偷懒、做错事、拒绝指令 |
| **NPC 自主性** | 被动等待指令 | AI 每隔一天自动规划全天行动，性格影响计划风格（勤劳排满/懒惰摸鱼） |
| **管理深度** | 简单的资源加减 | 村民体力/心情/性格/执行准确度多维管理 + 随时干预修改计划 |
| **经济** | 固定价格买卖 | 独立市场经济系统 + AI 分析师 + 实时价格走势 |
| **天气** | 固定/随机修饰 | AI 天气预报系统，影响农作物/村民行动/市场，需提前决策 |
| **事件** | 预设随机事件 | AI 根据天气 + 村庄状态生成情境化事件 |
| **交互乐趣** | 数值优化 | 和 AI 村民的性格碰撞与对话策略 |
| **AI 协同** | 单一 AI 模块 | 天气/市场/村民三大 AI 模块联动，信息流转，季末记忆压缩 |

### 1.4 AI 可移除性测试 ✅

> 移除 AI 后，游戏仍可作为传统面板经营游戏运行，但会失去：
> - 自然语言指挥村民的核心体验
> - 村民的独特性格反馈（偷懒、讨价还价等）
> - 村民个性化的每日行动计划（降级为固定模板计划）
> - 自然语言天气预报和市场简报（降级为固定文本模板）
> - 动态市场博弈
> - 情境化事件生成
>
> **降级方案**：若 LLM 不可用，系统自动切换为按钮指令 + 预设脚本对话模式

### 2.2 三层体验架构

```
┌─────────────────────────────────────────────────┐

│   (市场供需 / 贸易博弈 / 经济周期)               │
├─────────────────────────────────────────────────┤
│         第二层：AI 村民管理                       │
│   (性格 / 体力 / 对话指挥 / 自主决策)            │
├─────────────────────────────────────────────────┤

│   (房屋 / 农田 / 设施 / 资源生产)                │

│ 横切关注点：🌤️ 天气系统（影响全部三层）          │
│   天气 → 作物生长/村民行为/市场价格/体力消耗     │
│ 横切关注点：🤖 AI 上下文管理                     │

└─────────────────────────────────────────────────┘
```

### 2.3 玩家角色定义

玩家扮演**村长**，拥有以下权力和限制：

| 权力 | 限制 |
|-----|------|
| 招募/解雇村民 | 需要金币 + 空余房屋 |
| 通过对话分配任务 | 村民可能拒绝或打折执行 |
| 建造/升级建筑 | 需要资源和金币 |
| 市场买卖决策 | 价格由供需动态决定 |
| 根据天气预报安排农活 | 天气预报有误差，且村民对天气反应不同 |
| 限制/允许村民自主交易 | 村民可能不听话（叛逆性格） |
| 制定村庄发展方向 | 受限于当前资源、人力和天气 |

## 三、第一层：村庄基础建设系统

### 3.1 资源体系

游戏共有以下基础资源：

| 资源 | 图标 | 获取方式 | 用途 |
|-----|------|---------|------|
| **金币** | 💰 | 出售农产品/加工品 | 招募村民、购买种子、建造建筑 |
| **粮食** | 🌾 | 农田产出 | 村民食物消耗、出售 |
| **木材** | 🪵 | 伐木场产出 | 建造建筑 |
| **石料** | 🪨 | 采石场产出 | 建造建筑 |
| **种子** | 🌱 | 市场购买 | 种植作物 |

### 3.2 建筑体系

#### 房屋系统（核心限制）

房屋是招募村民的**前置条件**，决定了村庄可容纳的村民上限：

| 建筑 | 造价 | 容纳村民 | 说明 |
|-----|------|---------|------|
| **茅草屋** | 50💰 + 20🪵 | 1人 | 初始可建，简陋但便宜 |
| **木屋** | 120💰 + 50🪵 + 10🪨 | 2人 | 需要解锁 |
| **石屋** | 250💰 + 30🪵 + 60🪨 | 3人 | 需要解锁 |

> **核心规则**：
> - 一个农场（村庄）**最多同时拥有 4 名村民**帮忙干活
> - 想招募第 5 名村民？必须先**解雇**当前 4 名中的 1 位
> - 房屋总容量 ≥ 当前村民数，才能招募新村民
> - 解雇村民不退还招募费用（慎重决策！）

#### 生产建筑

| 类别 | 建筑 | 造价 | 产出 | 需分配村民 |
|-----|------|------|------|----------|
| **采集** | 农田 ×1（初始） | 免费 | 粮食/作物 | 1人 |
| **采集** | 农田（扩建） | 80💰 + 15🪵 | 粮食/作物 | 1人 |
| **采集** | 伐木场 | 100💰 + 30🪵 | 木材 | 1人 |
| **采集** | 采石场 | 100💰 + 20🪵 | 石料 | 1人 |
| **加工** | 磨坊 | 200💰 + 40🪵 + 20🪨 | 小麦→面粉 | 1人 |
| **加工** | 面包坊 | 300💰 + 50🪵 + 30🪨 | 面粉→面包 | 1人 |
| **市政** | 仓库（升级） | 150💰 + 30🪵 | 扩大存储上限（+50） | 无需 |
| **市政** | 水井 | 60💰 + 10🪨 | 减少农田浇水需求 | 无需 |

#### 仓库容量系统

所有资源都受**仓库容量限制**（金币除外），每种资源的存储上限 = **仓库总容量**，且**所有资源总量之和**不能超过仓库总容量。

- **基础仓库容量**：80
- **每次仓库升级**：+50（最多升级 3 次，满级仓库总容量 = 230）
- 资源面板改为**木桶堆叠显示**，堆叠高度反映占用比例

| 资源类型 | 存储上限 | 说明 |
|---------|---------|------|
| 💰 金币 | ∞ | 核心货币，**不占仓库容量** |
| 其他资源 | 仓库总容量 | 自由存储，单品上限一致，总量受仓库容量约束 |

> **设计理念**：自由存储降低早期管理门槛，单品上限清晰直观；通过升级仓库提高整体上限，仍保留扩容动机。

## 四、第二层：AI 村民系统（核心创新）

### 4.1 村民属性架构

每个村民由以下属性构成：

```
┌─────────────────────────────────────────────────────┐
│                   AI 村民属性卡                       │
├─────────────────────────────────────────────────────┤
│                                                     │
│  📛 基本信息                                        │
│  ├─ 姓名（系统随机生成）                             │
│  ├─ 头像（唯一 Emoji，如 👩‍🌾🧑‍🍳🦊🌻 等）           │
│  ├─ 年龄（影响体力上限和恢复速度）                    │
│  └─ 职业倾向（影响工作效率）                         │
│                                                     │
│  🧠 性格特征（招募时随机组合，2-3 个标签）            │
│  ├─ 勤劳 / 懒惰      （影响工作速度和主动性）        │
│  │   勤劳：任务执行快，空闲时主动找活干              │
│  │   懒惰：任务拖拖拉拉，经常磨洋工、找借口         │
│  ├─ 聪明 / 愚笨      （影响指令理解和执行准确度）    │
│  │   聪明：准确理解指令，还会主动提出优化建议        │
│  │   愚笨：可能误解指令，做错事，但老实本分          │
│  ├─ 听话 / 叛逆      （影响指令服从度）              │
│  │   听话：几乎不拒绝任务，执行力强                 │
│  │   叛逆：有自己想法，不合理指令会反驳或拒绝        │
│  ├─ 乐观 / 悲观      （影响心情和抗压能力）          │
│  │   乐观：心情恢复快，累了也笑着干                 │
│  │   悲观：容易抱怨，动不动就发牢骚                 │
│  └─ 健壮 / 体弱      （影响体力上限和消耗）          │
│      健壮：体力上限高，重活也不怕                    │
│      体弱：容易累，需要频繁休息                     │
│                                                     │
│  💪 体力系统                                        │
│  ├─ 当前体力 / 最大体力（如 6/10）                   │
│  ├─ 体力消耗速率（受性格影响）                       │
│  └─ 体力恢复方式（休息 or 消耗食物，AI 自主选择）     │
│                                                     │
│  😊 心情系统                                        │
│  ├─ 当前心情值（0-10）                               │
│  ├─ 影响因素：工作强度、村长态度、休息是否充足        │
│  └─ 心情过低 → 消极怠工、拒绝执行、甚至请求离开      │
│                                                     │
│  ⭐ 技能系统                                        │
│  ├─ 种植（影响农作物产量和质量）                     │
│  ├─ 采集（影响伐木/采石效率）                        │
│  ├─ 加工（影响加工品质量）                           │
│  └─ 技能通过反复执行对应任务提升                      │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 4.2 村民招募系统

#### 核心机制：属性预设池 + 随机组合

系统维护一个**预设属性池**，每次招募时从池中随机抽取并组合，生成独一无二的村民。组合结果确定后，这些属性将**永久作为该村民的 LLM 上下文**，驱动其所有对话和行为。

```
┌─────────────────────────────────────────────────────┐
│              属性预设池 + 随机组合机制                  │
├─────────────────────────────────────────────────────┤
│                                                     │
│  系统预设属性池：                                     │
│  ┌─────────────────────────────────────────────┐    │
│  │  性格池（互斥对，每对最多抽1个）              │    │
│  │  ├─ 勤劳 ↔ 懒惰                             │    │
│  │  ├─ 聪明 ↔ 愚笨                             │    │
│  │  ├─ 听话 ↔ 叛逆                             │    │
│  │  ├─ 乐观 ↔ 悲观                             │    │
│  │  └─ 健壮 ↔ 体弱                             │    │
│  │                                              │    │
│  │  特长池（抽1个）                              │    │
│  │  ├─ 种植能手 / 采集好手 / 加工巧匠           │    │
│  │                                              │    │
│  │  口癖池（抽1个，增加角色辨识度）              │    │
│  │  ├─ "嘿嘿" / "唉..." / "没问题！"           │    │
│  │  ├─ "这个嘛..." / "交给我吧" / "好累啊~"    │    │
│  │  └─ "报告村长！" / "随便啦" / "我尽力..."    │    │
│  └──────────────────────────────────────────────┘    │
│                                                     │
│  招募时组合流程（玩家不可见，后台执行）：              │
│  1. 从 5 对性格中随机抽取 2-3 个（不抽互斥对）       │
│  2. 从特长池随机抽取 1 个                            │
│  3. 从口癖池随机抽取 1 个                            │
│  4. 根据性格组合推算体力上限、准确率、工作速度        │
│  5. 随机生成姓名                                     │
│  6. 玩家只能看到姓名，确认招募后才揭晓全部属性       │
│                                                     │
│  ┌──────────────────────────────────────────┐       │
│  │  组合结果 → 永久绑定为该村民的 LLM 上下文  │       │
│  │  每次对话/行为决策时自动注入 Prompt         │       │
│  └──────────────────────────────────────────┘       │
│                                                     │
└─────────────────────────────────────────────────────┘
```

#### 招募流程（盲抽机制）

```
┌─────────────────────────────────────────────────────┐

│     └─ 村民自动入住空闲房屋                          │
│                                                     │
```

#### 属性揭晓时机

| 属性 | 何时可见 | 说明 |
|-----|---------|------|
| **姓名** | 招募完成后立即 | 第一时间知道叫什么 |
| **性格标签** | 招募完成后立即 | 揭晓动画中逐一展示 |
| **特长** | 招募完成后立即 | 揭晓动画中展示 |
| **口癖** | 首次对话时自然展现 | 不直接标注，靠对话发现 |
| **体力上限** | 招募完成后可见 | 显示在村民面板 |
| **准确率/速度** | 隐藏属性 | 不直接展示数值，靠实际表现观察 |

> **设计理念**：准确率和工作速度是**隐藏属性**——玩家不会直接看到"准确率 70%"这个数字，而是在实际指挥中**逐渐发现**："这家伙怎么老做错事？""为啥同样的活他干得这么慢？"通过体验感知村民个性，而非通过数值表格。

#### 固定费用

| 操作 | 费用 | 说明 |
|-----|------|------|
| **招募村民** | 50💰 | 固定费用，盲抽，无法预览属性 |
| **解雇村民** | 20💰 | 固定遣散费（低于招募费） |

#### 属性注入 LLM 上下文

村民被招募后，其所有属性将**永久作为该 NPC 的 LLM Prompt 上下文**：

```
招募时确定的属性：
├─ 性格标签（如：懒惰、聪明）     ──→ 注入 Prompt 的"性格行为指南"
├─ 特长（如：种植能手）           ──→ 注入 Prompt 的"能力描述"
├─ 口癖（如："好累啊~"）          ──→ 注入 Prompt 的"说话风格"
├─ 体力上限/准确率/速度           ──→ 注入 Prompt 的"当前状态"
└─ 姓名                          ──→ 注入 Prompt 的"角色名"

每次 LLM 调用时，完整属性包作为 System Prompt 的一部分，
确保 AI 的回复始终与该村民的人设一致。
```

#### 解雇机制

- 解雇村民需支付 **20💰** 遣散费（低于招募的 50💰）
- 解雇后其他村民心情 -1（觉得村长无情）
- 被解雇的村民永久离开，不会再出现在候选列表中
- 如果通过对话好好安抚其他村民，可减轻心情影响
- 腾出的房屋空位可立即用于招募新村民

> **注意**：村民招募后**不需要日薪**，只需要消耗食物（每日 1 份粮食）维持生存。管理成本体现在"食物供给"和"性格应对"上，而非金钱。

| 指令类型 | 示例语句 | 村民反馈 |
|---------|---------|---------|
| **种植** | "小青，去种一批萝卜" | 执行种植任务 |
| **浇水** | "帮我给农田浇浇水" | 执行浇水任务 |
| **施肥** | "把这块地施点肥" | 执行施肥任务 |

| **加工** | "把小麦磨成面粉" | 执行加工任务 |

#### 村民执行偏差的情况

任务下达后，村民**不一定按你想的方式执行**，这取决于性格：

| 情况 | 村民反应 | 性格触发 |
|-----|---------|---------|
| **拖延执行** | "嗯...等我歇会儿再去吧..." | 懒惰：任务延迟 1-3 小时才开始 |
| **做错事情** | "啊？你说的是浇水不是施肥？😅" | 愚笨：30% 概率执行错误任务 |
| **做一半就停** | "干了一半好累啊，明天再说吧~" | 懒惰+体力中等：任务可能半途而废 |
| **理解偏差** | "你说种萝卜？我种了土豆..." | 愚笨：可能种错作物或去错农田 |
| **直接拒绝** | "凭什么又是我？不公平！" | 叛逆：需要好言相劝或换人 |
| **磨洋工** | 接受任务但效率只有 50% | 懒惰+心情低：消极怠工 |
| **体力不足** | "村长，我真的没力气了..." | 任何性格，体力<3 时 |
| **夜间抗拒** | "都这么晚了还要干活？😴" | 非"勤劳"性格，夜间指令 |

> **关键设计**：愚笨的村民不是故意捣乱，而是理解能力差。村长需要用**更清晰简洁的指令**来降低出错概率，或者干脆把复杂任务交给聪明的村民。

### 4.5 村民自主行为

村民不只是被动等待指令，也会有**自主行为**：

| 自主行为 | 触发条件 | 说明 |
|---------|---------|------|
| **自动休息** | 体力 < 3 | AI 决定休息还是吃食物 |
| **闲逛摸鱼** | 无任务 + 性格"懒惰" | 不干活但也不消耗体力 |
| **主动干活** | 无任务 + 性格"勤劳" | 自动寻找未完成的农活 |
| **自作主张** | 无任务 + 性格"愚笨" | 可能自己乱干一通（如把成熟作物当杂草拔了） |
| **主动提建议** | 性格"聪明" + 发现问题 | "村长，我觉得可以..." |
| **向村长抱怨** | 心情 < 3 | 主动弹出对话请求 |
| **帮助同伴** | 性格"乐观" + 同伴体力低 | 分担一部分工作 |
| **请求离开** | 心情持续 < 2 超过 5 天 | 警告！可能自行离开 |

### 4.6 村民头像系统

每个村民拥有**独特的 Emoji 头像**，在所有界面（村民卡片、对话框、底部栏、聊天流、日志）中统一显示。

头像池包含人物类（👩‍🌾👨‍🌾🧑‍🍳👷等）、动物类（🦊🐱🐼等）、自然类（🌻🌸🍄等），系统在村民创建时自动分配未使用的头像。初始村民小青固定使用 👩‍🌾。

### 4.7 村民自动调度系统（核心机制）

系统**每日 7:00**（在市场早报 5:00 之后）自动为每个村民**并行调用** LLM，生成该村民**当天全天的行动计划**（计划从 8:00 开始执行，7:00-8:00 为起床准备时间）。调度上下文包含当天市场早报和昨日晚报。

#### 作息时间

| 时间 | 状态 |
|------|------|
| 7:00 | 起床，生成当日计划（7:00-8:00 为起床准备时间） |
| 8:00-21:59 | 按计划执行行动（8:00 开始第一个任务） |
| 22:00 | 睡觉（自动恢复体力） |

#### 设计原则

```
┌─────────────────────────────────────────────────────────────┐
│               村民调度系统：两种 LLM 调用模式                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  模式 A：系统自动调度（每日 7:00，并行调用）                 │
│  ├─ 为每个村民并行生成当天全天行动计划（Promise.all）       │
│  ├─ LLM 上下文包含：市场早报(5:00)+昨日晚报+村庄状态      │
│  ├─ LLM 输出严格 JSON，按时段安排固定行动类型               │
│  ├─ 市场交易(trade)必须安排在 9:00-17:59 之间              │
│  └─ 生成后，系统解析 JSON + 执行时做现实检查                │
│                                                             │
│  模式 B：玩家对话干预（随时触发）                            │
│  ├─ 玩家主动与村民对话，可改变/覆盖当前计划                 │
│  ├─ LLM 输出对话回复 + 可能的计划修改                       │
│  └─ 仅玩家主动对话时游戏暂停                                │
│                                                             │
│  执行时现实检查（计划 ≠ 现实）：                             │
│  ├─ 计划收获 → 但作物未成熟 → 跳过(⚠️)                     │
│  ├─ 计划买东西 → 但金币不足 → 跳过(⚠️)                     │
│  ├─ 计划交易 → 但市场关门 → 跳过(⚠️)                       │
│  ├─ 体力不足 → 自动休息恢复 → 恢复后继续当前时段计划       │
│  └─ 粮食不足 → 无法吃饭 → 跳过(⚠️)                        │
│                                                             │
│  调度时间线示例：                                            │
│  ─────────────────────────────────────────────               │
│  每日 5:00  市场早报发布                                     │
│  每日 7:00  村民起床 → 并行生成当天计划                      │
│  7:00-8:00  起床准备时间                                     │
│  8:00-21:59 村民按计划执行（含现实检查）                     │
│  22:00      村民睡觉                                         │
│  ─────────────────────────────────────────────               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

| `pest_control` | 除虫 | 🐛 | 2h | 2 | 虫害天气期间 | 手动除虫（减少产量损失） |
| `chop` | 伐木 | 🪓 | 2h | 2 | 有伐木场 | 获取木材 |
| `mine` | 采石 | ⛏️ | 2h | 3 | 有采石场 | 获取石料 |
| `process` | 加工 | 🏭 | 3h | 2 | 有加工坊 | 原料加工为成品 |
| `trade` | 市场交易 | 🛒 | 1h | 1 | 市场开放(9:00-18:00) | 买入或卖出物品 |

#### 系统自动调度的 LLM Prompt

调度 Prompt 的关键上下文：

| 上下文数据 | 来源 | 说明 |
|-----------|------|------|
| 市场早报 | MarketAnalyst 5:00 | 今日走势预测和建议 |
| 昨日晚报 | MarketAnalyst 前日 19:00 | 昨日交易回顾 |
| 农田状态 | FarmSystem | 含作物进度百分比，判断是否成熟 |
| 其他村民计划 | 已生成的村民 | 避免重复安排同类任务 |

#### 系统调度解析与执行

```javascript
// js/systems/VillagerScheduler.js
class VillagerScheduler {
    constructor(gameState, eventBus, aiService) {
        this.state = gameState;
        this.bus = eventBus;
        this.ai = aiService;

        // 每个村民的调度状态
        // key: villagerId, value: { schedule, targetDay, lastScheduleDay, cancelledByPlayer }
        this.schedules = {};

        this.bus.on('newDay', (data) => this.onNewDay(data));
        this.bus.on('tick', (data) => this.onTick(data));
    }

    // ========== 系统自动调度（每隔 1 天，清晨 4:00） ==========
    async onNewDay(data) {
        for (const villager of this.state.villagers) {
            const sched = this.schedules[villager.id];

            // 判断是否需要调度（每隔 1 天，即上次调度后过了 1 天）
            const lastDay = sched?.lastScheduleDay || 0;
            const shouldSchedule = (data.day - lastDay) >= 2;

            // 如果被玩家干预取消了本次调度，跳过
            if (sched?.cancelledByPlayer) {
                this.schedules[villager.id].cancelledByPlayer = false;
                continue;
            }

            if (shouldSchedule) {
                await this.generateSchedule(villager, data.day + 1); // 生成明天的计划
            }
        }
    }

    // 生成某个村民的次日行动计划
    async generateSchedule(villager, targetDay) {
        const context = this.buildScheduleContext(villager, targetDay);
        const prompt = generateSchedulePrompt(context);

        const result = await this.ai.chat(prompt, { temperature: 0.7, maxTokens: 600 });

        if (result && result.schedule) {
            // 验证 schedule 合法性
            const validated = this.validateSchedule(result.schedule, villager, targetDay);
            this.schedules[villager.id] = {
                schedule: validated,
                targetDay,
                lastScheduleDay: targetDay - 1,
                thought: result.thought || '',
                currentIndex: 0,
                cancelledByPlayer: false,
            };
            this.bus.emit('scheduleReady', { villager, schedule: validated, targetDay });
        } else {
            // LLM 失败时降级：生成默认计划
            this.schedules[villager.id] = {
                schedule: this.fallbackSchedule(villager),
                targetDay,
                lastScheduleDay: targetDay - 1,
                thought: '（系统默认计划）',
                currentIndex: 0,
                cancelledByPlayer: false,
            };
        }
    }

    // 每 Tick 推进村民执行计划
    onTick(data) {
        const hour = this.state.time.hour;
        const day = this.state.time.day;

        for (const villager of this.state.villagers) {
            const sched = this.schedules[villager.id];
            if (!sched || sched.targetDay !== day) continue;

            // 找到当前时段应该执行的行动
            const currentAction = sched.schedule.find(
                s => hour >= s.startHour && hour < s.startHour + s.duration
            );

            if (currentAction && villager.currentAction !== currentAction.action) {
                this.executeAction(villager, currentAction);
            }
        }
    }

    // 验证 LLM 输出的合法性
    validateSchedule(schedule, villager, targetDay) {
        const VALID_ACTIONS = [
            'plant','water','fertilize','harvest','pest_control',
            'chop','mine','process','trade','rest','eat','idle','chat'
        ];
        const weatherEffects = this.state.weather.getCurrentEffects?.() || {};

        return schedule.filter(s => {
            if (!VALID_ACTIONS.includes(s.action)) return false;   // 非法行动
            if (s.startHour < 6 || s.startHour > 20) return false; // 超出工作时间
            if (s.duration < 1 || s.duration > 4) return false;    // 时长异常
            // 天气禁止外出时，过滤户外行动
            if (!weatherEffects.canGoOut) {
                const outdoor = ['plant','water','harvest','chop','mine','fertilize'];
                if (outdoor.includes(s.action)) return false;
            }
            return true;
        });
    }

    // 降级默认计划
    fallbackSchedule(villager) {
        return [
            { startHour: 6,  action: 'water',   duration: 2, target: null, note: '浇水' },
            { startHour: 8,  action: 'harvest', duration: 2, target: null, note: '收获' },
            { startHour: 10, action: 'rest',    duration: 2, target: null, note: '休息' },
            { startHour: 12, action: 'eat',     duration: 1, target: null, note: '吃饭' },
            { startHour: 13, action: 'chop',    duration: 2, target: null, note: '伐木' },
            { startHour: 15, action: 'rest',    duration: 2, target: null, note: '休息' },
        ];
    }
}
```

```javascript
// 玩家与村民对话时的冲突检测
function onPlayerChat(villager, playerInput) {
    const hour = gameState.time.hour;
    const day = gameState.time.day;
    const SCHEDULE_HOUR = 4;       // 系统调度时间
    const THRESHOLD_HOURS = 4;     // 冲突阈值

    // 计算距下次调度的小时数
    const hoursToNextSchedule = (hour < SCHEDULE_HOUR)
        ? (SCHEDULE_HOUR - hour)                    // 同一天
        : (24 - hour + SCHEDULE_HOUR);              // 跨天

    const isNearScheduleTime = hoursToNextSchedule <= THRESHOLD_HOURS;

    // 构建对话上下文
    const context = buildVillagerContext(villager);
    context.mode = 'playerChat';
    context.isNearScheduleTime = isNearScheduleTime;
    context.nextDay = day + (hour >= SCHEDULE_HOUR ? 1 : 0) + 1;

    if (isNearScheduleTime) {
        // 标记取消该村民的下次系统调度
        scheduler.schedules[villager.id].cancelledByPlayer = true;
        context.additionalInfo = `现在接近次日，这次对话同时规划明天的安排。`;
    }

    // 调用 LLM
    const prompt = generateVillagerPrompt(context);
    return aiService.chat(prompt);
}
```

### 5.2 数学模型：100 个虚拟交易者

#### 价格引擎核心公式

系统采用**均值回归 + 随机游走**模型，保证价格有波动但整体稳定：

```javascript
// 每 Tick（每小时）更新一次市场价格
function updateMarketPrice(item, tick) {
    const config = MARKET_ITEMS[item];

    // === 1. 均值回归力（拉向基准价，保持稳定） ===
    const deviation = (currentPrice - config.basePrice) / config.basePrice;
    const meanReversion = -deviation * config.reversionSpeed;
    // reversionSpeed = 0.02~0.05，偏离越大拉力越强

    // === 2. 随机游走（模拟市场噪声） ===
    const noise = (Math.random() - 0.5) * config.volatility;
    // volatility = 0.01~0.05，控制波动幅度

    // === 3. 虚拟交易者供需压力 ===
    const traderPressure = simulateTraders(item, currentPrice, config);
    // 100个虚拟交易者的净买卖方向

    // === 4. 季节修正 ===
    const seasonMod = getSeasonModifier(item, currentSeason);
    // 春季种子贵，秋季粮食便宜

    // === 5. 事件冲击（临时，会衰减） ===
    const eventShock = getActiveEventShock(item);
    // 虫灾→粮价涨，丰收→粮价跌

    // === 6. 玩家/村民交易影响 ===
    const playerImpact = getRecentTradeImpact(item);
    // 玩家大量抛售 → 价格下压

    // === 综合计算 ===
    const priceChange = meanReversion + noise + traderPressure
                      + seasonMod + eventShock + playerImpact;

    currentPrice = currentPrice * (1 + priceChange);

    // === 硬限制：价格不会偏离基准太远 ===
    currentPrice = clamp(currentPrice,
        config.basePrice * 0.3,
        config.basePrice * 3.0
    );

    return currentPrice;
}
```

#### 虚拟交易者模拟

100 个虚拟交易者分为 5 种类型，模拟不同的市场参与者：

| 类型 | 数量 | 行为模式 | 对价格的影响 |
|-----|------|---------|------------|
| **日常消费者** | 40人 | 稳定买入粮食/日用品，需求随季节变化 | 提供基础需求支撑 |
| **农场主** | 25人 | 定期卖出农产品，收获季集中出货 | 收获季造成供给冲击 |
| **投机商人** | 15人 | 低买高卖，追涨杀跌 | 放大短期波动 |
| **囤货商** | 10人 | 价格低时大量买入囤积，高时抛售 | 提供价格缓冲 |
| **随机散户** | 10人 | 随机小额买卖 | 增加市场噪声 |

```javascript
// 模拟100个虚拟交易者的综合影响
function simulateTraders(item, price, config) {
    let netPressure = 0;
    const ratio = price / config.basePrice;  // 当前价/基准价

    // 日常消费者：价格越高买得越少
    const consumerDemand = 40 * Math.max(0, 1 - (ratio - 1) * 0.5);
    netPressure += consumerDemand * 0.001;

    // 农场主：总是卖出，收获季卖得更多
    const farmerSupply = 25 * (isHarvestSeason() ? 1.8 : 1.0);
    netPressure -= farmerSupply * 0.001;

    // 投机商人：追涨杀跌（动量策略）
    const momentum = getPriceTrend(item, 24); // 近24小时趋势
    const specPressure = 15 * momentum * 0.8;
    netPressure += specPressure * 0.001;

    // 囤货商：逆势操作（均值回归策略）
    const hoarderPressure = 10 * (1 - ratio) * 0.5; // 便宜就买，贵就卖
    netPressure += hoarderPressure * 0.001;

    // 随机散户
    netPressure += (Math.random() - 0.5) * 10 * 0.001;

    return netPressure;
}
```

> **关键规则**：村民用的是**村庄公共库存和金币**进行交易，所以愚笨村民乱买卖可能会亏钱！这是"用人风险"的一部分——你是否敢让一个笨蛋在空闲时接触市场？
>
> **村长干预**：如果不想让某个村民自主交易，可以通过对话明确告知："你不许去市场乱买东西！"（听话的村民会遵守，叛逆的…不一定）

## 五·二、繁荣度系统

### 核心机制
- 繁荣度为**累计制**，**可增可减**（最低为 0），无上限
- 每日根据村庄正面状态**自动增长**，同时负面状态会导致**衰减**
- **衰减力度相对增长力度较小**，鼓励玩家维护村庄健康状态
- 分 **10 个等级区间**，达到相应等级可**手动领取金币奖励**
- 繁荣度降低可能导致**降级**（但已领取的奖励不会被收回）
- 达到 Lv.10「👑 传说桃源」即为通关

### 每日自动增长（📈 正面因素）

| 来源 | 增长量 | 说明 |
|------|--------|------|
| 👥 每名村民 | +1/天 | 满编4人每天+4 |
| 🏗️ 每座建筑 | +0.5/天 | 向下取整 |
| 🌾 每块活跃农田 | +0.5/天 | 有作物才算 |
| 😊 平均心情≥6 | +1/天 | ≥8则+2/天 |
| 💰 金币≥200 | +1/天 | — |
| 🌾 粮食≥20 | +1/天 | — |

> 有村民时每日最少增长 1 点。

### 每日自动衰减（📉 负面因素，力度较小）

| 原因 | 衰减量 | 触发条件 |
|------|--------|---------|
| 😞 士气低落 | -1/天 | 村民平均心情 < 3 |
| 😡 民怨沸腾 | -2/天 | 村民平均心情 < 2（与上条叠加为 -2） |
| 🍚 饥荒 | -2/天 | 粮食为 0 且有村民 |
| 💸 财政困难 | -1/天 | 金币为 0 |
| 👻 人口空虚 | -1/天 | 无村民 |
| 🏜️ 田地荒废 | -1/天 | 有农田但全部无作物 |

> 每日净变化 = 增长 - 衰减。最坏情况下日衰减约 -7（同时触发全部负面），但正常经营下增长远大于衰减。

### 事件加成/扣减

| 事件 | 变化 | 类型 |
|------|------|------|
| 🏗️ 建造新建筑 | +5 | 加成 |
| 👥 招募新村民 | +10 | 加成 |
| 🌾 收获作物 | +2 | 加成 |
| 💔 村民离开 | -5 | 扣减 |
| 🥀 作物枯萎 | -1 | 扣减 |

### 等级与奖励

| 等级 | 名称 | 所需繁荣度 | 金币奖励 | 图标 |
|------|------|-----------|---------|------|
| Lv.1 | 荒芜村落 | 0 | — | 🏚️ |
| Lv.2 | 初建小村 | 20 | 50💰 | 🏠 |
| Lv.3 | 安宁村庄 | 60 | 100💰 | 🏡 |
| Lv.4 | 朝气小镇 | 120 | 200💰 | 🌱 |
| Lv.5 | 繁忙集市 | 200 | 300💰 | 🛒 |
| Lv.6 | 富饶之地 | 300 | 500💰 | 🌾 |
| Lv.7 | 兴旺村镇 | 450 | 800💰 | 🏘️ |
| Lv.8 | 锦绣乡里 | 650 | 1200💰 | 🌸 |
| Lv.9 | 四海升平 | 900 | 2000💰 | 🏛️ |
| Lv.10 | 传说桃源 | 1200 | 3000💰 | 👑 |

> **设计理念**：繁荣度可增可减，给村庄经营带来紧迫感和策略深度。衰减力度明显低于增长力度，鼓励积极经营而非惩罚玩家——只有严重疏忽管理（饥荒、全员不满、财政破产）才会产生显著衰减。等级可升可降，但已领取的奖励不会被收回，保证玩家不会完全"白干"。非线性的阈值递增（20→60→120→...→1200）让后期需要更长时间积累，保证游戏深度。

```

☀️ 早报 (5:00)        — ☀️ 市场早报（走势预测）

                         7:00-8:00 起床准备时间
📋 计划执行 (8:00)    — 村民从 8:00 开始按计划执行行动

🏪 市场开放 (9:00)    — 🟢 市场开门，可进行买卖交易
☀️ 上午 (9:00-12:00)  — 工作高峰期，村民按计划执行行动（含现实检查），市场交易活跃
🍽️ 午间 (12:00-14:00) — 村民按计划执行（可能安排 eat/rest/idle）
🏪 市场关闭 (18:00)   — 🔴 市场关门
🌙 晚报 (19:00)       — 🌙 市场晚报（交易回顾 + 玩家点评）
🌆 傍晚 (18:00-22:00) — 收工时间，适合对话/总结
😴 睡觉 (22:00)       — 村民睡觉（自动恢复体力），可选跳过
```

### 6.4 每 Tick 执行循环

```
每 Tick 执行：
├─ 1. 时间推进（小时/天/季/年）
├─ 2. 天气系统更新（每日触发）
│     ├─ 特殊天气倒计时，到期则回到季节默认
│     ├─ 检查今日是否有预定的特殊天气事件 → 弹窗通知
│     └─ 应用当前天气效果（季节默认 or 特殊天气的固定效果）
├─ 3. 作物生长更新（受天气效果影响）
│     ├─ 生长速度 × 当前天气 cropGrowth 系数
│     ├─ 特殊天气：判定作物受损/枯萎/自动浇水等
│     └─ 春雨连绵：自动浇水
├─ 4. 村民调度与状态更新
│     ├─ 4a. 每日 7:00 系统自动调度（并行调用 Promise.all）
│     │     ├─ 上下文包含市场早报(5:00) + 昨日晚报 + 村庄状态
│     │     ├─ 并行为每个村民调用 LLM 生成当日行动计划
│     │     ├─ 验证：行动枚举、时间范围(8:00-21:59)、市场时间(trade仅9-17时)
│     │     ├─ 若 LLM 失败 → 降级使用默认计划
│     │     └─ 若被玩家干预取消 → 跳过本次调度
│     ├─ 4b. 按计划执行（每 Tick 检查 + 现实检查）
│     │     ├─ 查找当前时段对应的行动，执行前做现实检查
│     │     ├─ 现实检查：作物未成熟/金币不足/市场关闭 → 跳过(⚠️)
│     │     ├─ 体力不足 → 强制休息恢复 → 恢复后继续当前时段计划
│     │     └─ 22:00 村民睡觉，8:00 前不执行计划行动（7:00-8:00 起床准备）
│     ├─ 4c. NPC 自由对话（每 Tick 检查预排表）
│     │     ├─ 0:00 预排发言顺序和时间点（每人每天最多 2 次，间隔≥2h）
│     │     ├─ 串行调用 LLM，保证对话上下文连贯
│     │     └─ 降级时使用性格对应的预设发言
│     └─ 4d. 心情衰减（恶劣天气额外影响悲观村民）
├─ 5. 市场价格更新（每 Tick）
├─ 6. AI 模块调用（定时触发）
│     ├─ 每5天/季初：天气预测 AI → 生成 5 天时间表
│     ├─ 每日 7:00：村民调度 AI → 并行生成当日计划（见 4a）
│     ├─ 5:00 天气每日播报 LLM → 简短播报（后台，不暂停）
│     ├─ 5:00 市场分析师 LLM → ☀️ 早报（走势预测，后台不暂停）
│     ├─ 8:00-21:00 NPC 自由对话 LLM → 串行调用（后台不暂停）
│     ├─ 19:00 市场分析师 LLM → 🌙 晚报（交易回顾+玩家点评，后台不暂停）
│     └─ 季末：上下文压缩总结（村民记忆/市场/天气）
├─ 7. 经济结算（每日触发）
│     ├─ 扣除村民食物消耗（每人每日 1 份粮食）
│     ├─ 雪天额外扣除取暖木材（1🪵/天）
│     └─ 检查粮食库存是否充足
├─ 8. 事件检测
│     ├─ 天气事件（干旱、暴雨损害等）
│     ├─ 随机事件触发
│     └─ 村民主动行为触发（含自主交易、天气反应）
└─ 9. UI 更新（节流，每 4 Tick 批量更新）
```

### 6.5 自动暂停规则

游戏在以下情况会自动暂停，防止玩家错过重要事件：

- 作物成熟时
- **特殊天气事件触发时**（AI 预定的特殊天气到来，弹窗预警 + 效果说明）
- **玩家主动与村民 NPC 对话时**（对话期间暂停，结束后自动恢复）
- 村民拒绝执行任务时
- 重要事件触发时（虫灾、村民请求离开等）
- 金币/粮食即将耗尽时
- 繁荣度达到最终等级（通关）时

**以下情况不会暂停**（后台运行）：
- AI 市场分析师早报/晚报生成
- AI 天气预报播报
- AI 村民行动计划生成
- AI 每日总结生成
- 即时交易点评弹窗

## 七、事件系统

### 7.1 事件类型

#### 经济事件

| 事件 | 触发条件 | 选项 | 后果 |
|-----|---------|------|-----|
| 市场异动 | 单商品价格偏离基准>30% | 买入/卖出/观望 | 抓住差价机会或规避风险 |

| 村民乱交易 | 愚笨村民自主交易亏钱 | 训斥/禁止/算了 | 止损或放任 |

#### 村民事件

| 事件 | 触发条件 | 选项 | 后果 |

| 请求离开 | 心情持续极低 | 挽留/同意/好言相劝 | 可能失去村民 |

### 8.3 视觉风格指南

#### 设计理念

「治村物语」的 UI 风格应传达**清爽、温馨、田园**的感觉，让玩家仿佛置身于一个阳光明媚的小村庄。整体以浅色为主、草绿/麦黄/天蓝为点缀，呼应种田、村庄管理的主题。

#### 设计规范表

| 设计要素 | 规范 |
|---------|------|
| **整体色调** | 浅色田园风（米白底色、白色卡片），草绿/麦金/天蓝点缀 |
| **季节变化** | 面板底色和装饰条随季节微调（春粉、夏绿、秋金、冬蓝） |
| **字体** | 中文无衬线体（思源黑体），数字等宽（JetBrains Mono） |
| **图标** | Emoji 图标为主 + 简洁线条图标辅助 |
| **卡片** | 白底圆角（12px），轻微阴影，悬停时阴影加深 |
| **面板** | 米白/浅灰绿底色，浅棕边框，面板间保持 12px 间距 |
| **顶栏** | 深草绿底色 + 白色文字，传达"村长办公桌"的稳重感 |
| **按钮** | 草绿色=积极操作，麦金色=特殊操作，红色=危险操作，灰棕=次要 |
| **进度条** | 圆角条带（绿=健康≥60%，黄=警告30%-60%，红=危险<30%） |
| **数据变化** | 绿色↑=增长，红色↓=下降，灰色=不变 |
| **布局** | 三栏式：左侧资源/村民，中央操作区，右侧概览/日志 |
| **间距节奏** | 4px 基准：内边距 12/16px，卡片间距 12px，区块间距 20px |
| **动效** | 轻柔缓动（0.2s ease），避免花哨动画，体现田园慢节奏 |

```javascript
// js/core/EventBus.js
class EventBus {
    constructor() {
        this.listeners = {};
    }

    on(event, callback) {
        if (!this.listeners[event]) this.listeners[event] = [];
        this.listeners[event].push(callback);
    }

    off(event, callback) {
        if (!this.listeners[event]) return;
        this.listeners[event] = this.listeners[event].filter(cb => cb !== callback);
    }

    emit(event, data) {
        if (!this.listeners[event]) return;
        this.listeners[event].forEach(cb => cb(data));
    }
}

const eventBus = new EventBus();
```

```javascript
// ========== 预设属性池 ==========

const TRAIT_POOL = {
    positive: ['勤劳', '聪明', '听话', '健壮', '乐观'],
    negative: ['懒惰', '愚笨', '叛逆', '体弱', '悲观'],
};

const EXCLUSIVE_TRAITS = [
    ['勤劳', '懒惰'],
    ['聪明', '愚笨'],
    ['听话', '叛逆'],
    ['健壮', '体弱'],
    ['乐观', '悲观'],
];

const SPECIALTY_POOL = ['种植能手', '采集好手', '加工巧匠'];

const QUIRK_POOL = [
    '嘿嘿', '唉...', '没问题！', '这个嘛...',
    '交给我吧', '好累啊~', '报告村长！', '随便啦', '我尽力...',
];

// 固定费用
const RECRUIT_COST = 50;   // 招募费（盲抽）
const DISMISS_COST = 20;   // 解雇费

// ========== 生成候选村民（从预设池随机组合） ==========

function generateCandidate() {
    // 1. 从5对性格中随机抽取 2-3 个标签
    const traitCount = Math.random() < 0.4 ? 3 : 2;  // 40%概率3个，60%概率2个
    const traits = [];
    const usedPairs = new Set();  // 记录已使用的互斥对

    const allTraits = [...TRAIT_POOL.positive, ...TRAIT_POOL.negative];

    while (traits.length < traitCount) {
        const candidate = allTraits[Math.floor(Math.random() * allTraits.length)];

        // 检查互斥：找到该标签所属的互斥对
        const pairIndex = EXCLUSIVE_TRAITS.findIndex(pair => pair.includes(candidate));
        if (pairIndex !== -1 && usedPairs.has(pairIndex)) continue;  // 已用过这对
        if (traits.includes(candidate)) continue;  // 已抽过

        traits.push(candidate);
        if (pairIndex !== -1) usedPairs.add(pairIndex);
    }

    // 2. 随机抽取特长
    const specialty = SPECIALTY_POOL[Math.floor(Math.random() * SPECIALTY_POOL.length)];

    // 3. 随机抽取口癖
    const quirk = QUIRK_POOL[Math.floor(Math.random() * QUIRK_POOL.length)];

    // 4. 根据性格推算数值属性
    const maxStamina = traits.includes('健壮') ? 12
                     : traits.includes('体弱') ? 8 : 10;

    const accuracy = traits.includes('愚笨') ? 0.7
                   : traits.includes('聪明') ? 1.0 : 0.9;

    const workSpeed = traits.includes('懒惰') ? 0.6
                    : traits.includes('勤劳') ? 1.3 : 1.0;

    // 5. 生成完整村民对象（这些属性将永久注入 LLM 上下文）
    return {
        id: 'villager_' + Date.now() + Math.random().toString(36).substr(2, 4),
        name: generateRandomName(),

        // === 以下全部注入 LLM Prompt 上下文 ===
        traits,          // 性格标签（如 ['懒惰', '聪明']）
        specialty,       // 特长（如 '种植能手'）
        quirk,           // 口癖（如 '好累啊~'）

        // === 由性格推算的数值属性 ===
        stamina: maxStamina,
        maxStamina,
        mood: 7,
        accuracy,        // 指令理解准确率 (0.7-1.0)
        workSpeed,       // 工作速度倍率 (0.6-1.3)

        // === 可成长属性 ===
        skills: { farming: 1, gathering: 1, processing: 1 },
        currentTask: null,
    };
}

// 招募：盲抽一名村民（玩家不可预览属性）
function recruitVillager(gameState) {
    if (gameState.resources.gold < RECRUIT_COST) {
        return { success: false, reason: '金币不足' };
    }
    if (gameState.villagers.length >= gameState.MAX_VILLAGERS) {
        return { success: false, reason: '村民已满 4 人' };
    }
    if (gameState.villagers.length >= gameState.housingCapacity) {
        return { success: false, reason: '没有空余房屋' };
    }

    // 扣费
    gameState.resources.gold -= RECRUIT_COST;

    // 盲抽生成
    const villager = generateCandidate();
    gameState.villagers.push(villager);

    return { success: true, villager };  // 返回结果用于揭晓动画
}

// 解雇：需支付遣散费
function dismissVillager(gameState, villagerId) {
    if (gameState.resources.gold < DISMISS_COST) {
        return { success: false, reason: '金币不足以支付遣散费' };
    }

    const index = gameState.villagers.findIndex(v => v.id === villagerId);
    if (index === -1) return { success: false, reason: '村民不存在' };

    gameState.resources.gold -= DISMISS_COST;
    const dismissed = gameState.villagers.splice(index, 1)[0];

    // 其他村民心情 -5
    gameState.villagers.forEach(v => { v.mood = Math.max(0, v.mood - 5); });

    return { success: true, dismissed };
}

// 体力恢复 AI 决策
function decideStaminaRecovery(villager, gameState) {
    const { traits, stamina, maxStamina } = villager;
    const foodStock = gameState.resources.food;

    // 基础倾向分数（正=吃食物，负=休息）
    let eatScore = 0;

    if (traits.includes('勤劳')) eatScore += 30;     // 勤劳倾向吃食物继续干
    if (traits.includes('懒惰')) eatScore -= 40;     // 懒惰倾向休息（能歇就歇）
    if (traits.includes('聪明')) {                    // 聪明根据库存理性判断
        eatScore += foodStock > 20 ? 20 : -20;
    }
    if (traits.includes('愚笨')) {                    // 愚笨随机决定
        eatScore += Math.random() > 0.5 ? 15 : -15;
    }
    if (foodStock < 5) eatScore -= 50;               // 食物不够倾向休息
    if (villager.currentTask) eatScore += 20;        // 有未完成任务倾向吃

    if (eatScore > 0 && foodStock >= 1) {
        return { type: 'eat', reason: generateEatReason(villager) };
    } else {
        return { type: 'rest', reason: generateRestReason(villager) };
    }
}
```

```javascript
// js/core/TimeSystem.js
class TimeSystem {
    constructor(gameState, eventBus) {
        this.state = gameState;
        this.bus = eventBus;
        this.timer = null;
        this.BASE_INTERVAL = 3000; // 3秒 = 1游戏小时（1倍速）
    }

    start() {
        this.scheduleNextTick();
    }

    scheduleNextTick() {
        if (this.state.time.isPaused) return;

        const interval = this.BASE_INTERVAL / this.state.time.speed;
        this.timer = setTimeout(() => {
            this.tick();
            this.scheduleNextTick();
        }, interval);
    }

    tick() {
        const time = this.state.time;
        time.hour++;

        if (time.hour >= 24) {
            time.hour = 0;
            time.day++;
            this.bus.emit('newDay', { day: time.day });

            if (time.day > 28) {
                time.day = 1;
                time.month++;
                this.bus.emit('newMonth', { month: time.month });

                if (time.month > 12) {
                    time.month = 1;
                    time.year++;
                    this.bus.emit('newYear', { year: time.year });
                }
            }
        }

        this.bus.emit('tick', { hour: time.hour });
    }

    pause() {
        this.state.time.isPaused = true;
        clearTimeout(this.timer);
    }

    resume() {
        this.state.time.isPaused = false;
        this.scheduleNextTick();
    }

    setSpeed(speed) {
        this.state.time.speed = speed;
        clearTimeout(this.timer);
        if (!this.state.time.isPaused) this.scheduleNextTick();
    }
}
```

```javascript
// js/market/MarketEngine.js
class MarketEngine {
    constructor(gameState, eventBus) {
        this.state = gameState;
        this.bus = eventBus;
        this.prices = {};         // 当前价格 { itemId: price }
        this.history = {};        // 价格历史 { itemId: [{ tick, price }] }
        this.dailyReport = null;  // 今日 AI 分析报告

        // 初始化所有商品价格
        for (const [id, config] of Object.entries(MARKET_ITEMS)) {
            this.prices[id] = config.basePrice;
            this.history[id] = [];
        }

        // 监听 Tick：每小时更新价格
        this.bus.on('tick', () => this.updatePrices());
        // 监听新的一天：清晨生成分析报告
        this.bus.on('newDay', () => this.generateDailyReport());
    }

    // 每 Tick 更新所有商品价格
    updatePrices() {
        for (const [id, config] of Object.entries(MARKET_ITEMS)) {
            const price = this.prices[id];
            const deviation = (price - config.basePrice) / config.basePrice;

            // 均值回归 + 随机波动 + 虚拟交易者 + 季节
            const meanReversion = -deviation * config.reversionSpeed;
            const noise = (Math.random() - 0.5) * config.volatility;
            const traderPressure = this.simulateTraders(id, price, config);
            const seasonMod = this.getSeasonMod(id);

            const change = meanReversion + noise + traderPressure + seasonMod;
            this.prices[id] = Math.max(
                config.basePrice * 0.3,
                Math.min(config.basePrice * 3.0, price * (1 + change))
            );

            // 记录历史
            this.recordHistory(id, this.prices[id]);
        }
        this.bus.emit('marketUpdate', { prices: this.prices });
    }

    // 玩家/村民交易：影响价格
    executeTrade(itemId, quantity, isBuy) {
        const config = MARKET_ITEMS[itemId];
        const impact = (quantity / 10) * 0.03 * (isBuy ? 1 : -1);
        this.prices[itemId] *= (1 + impact);
        this.prices[itemId] = Math.max(
            config.basePrice * 0.3,
            Math.min(config.basePrice * 3.0, this.prices[itemId])
        );
        return this.prices[itemId];
    }

    // 获取 7 日趋势（用于 AI 分析和走势图）
    getTrend(itemId, hours = 168) {
        const hist = this.history[itemId];
        if (hist.length < 2) return 0;
        const recent = hist.slice(-hours);
        return (recent[recent.length - 1].price - recent[0].price) / recent[0].price;
    }
}
```

### 9.5 AI 调用设计

```javascript
// js/ai/AIService.js
class AIService {
    constructor(config) {
        this.apiUrl = config.apiUrl;       // OpenAI / Ollama 地址
        this.apiKey = config.apiKey;
        this.model = config.model || 'gpt-4o-mini';
        this.timeout = config.timeout || 8000;
    }

    async chat(prompt, options = {}) {
        try {
            const controller = new AbortController();
            const timeoutId = setTimeout(() => controller.abort(), this.timeout);

            const response = await fetch(this.apiUrl + '/v1/chat/completions', {
                method: 'POST',
                headers: {
                    'Content-Type': 'application/json',
                    'Authorization': `Bearer ${this.apiKey}`,
                },
                body: JSON.stringify({
                    model: this.model,
                    messages: [{ role: 'user', content: prompt }],
                    temperature: options.temperature || 0.8,
                    max_tokens: options.maxTokens || 500,
                }),
                signal: controller.signal,
            });

            clearTimeout(timeoutId);

            const data = await response.json();
            const content = data.choices[0].message.content;

            // 提取 JSON（兼容 markdown code block）
            const jsonMatch = content.match(/```json\s*([\s\S]*?)```/) 
                           || content.match(/\{[\s\S]*\}/);
            if (jsonMatch) {
                return JSON.parse(jsonMatch[1] || jsonMatch[0]);
            }

            throw new Error('无法解析 AI 响应');

        } catch (error) {
            console.warn('[AI] 调用失败，使用降级方案:', error.message);
            return null; // 返回 null 表示需要降级
        }
    }
}
```

```javascript
        const prompt = generateSchedulePrompt(context);

        const result = await this.ai.chat(prompt, { temperature: 0.7, maxTokens: 600 });

        if (result && result.schedule) {
            const validated = this.scheduler.validateSchedule(result.schedule, villager, targetDay);
            return { schedule: validated, thought: result.thought, staminaEstimate: result.staminaEstimate };
        }
        return null; // 触发降级
    }

    // 模式 B：玩家对话（随时调用）
    async handlePlayerChat(villager, playerInput, gameState) {
        const hour = gameState.time.hour;
        const SCHEDULE_HOUR = 4;
        const THRESHOLD = 4;
        const hoursToSchedule = (hour < SCHEDULE_HOUR)
            ? (SCHEDULE_HOUR - hour)
            : (24 - hour + SCHEDULE_HOUR);
        const isNearSchedule = hoursToSchedule <= THRESHOLD;

        const context = this.buildChatContext(villager, playerInput, gameState);
        context.isNearScheduleTime = isNearSchedule;

        if (isNearSchedule) {
            const nextDay = gameState.time.day + (hour >= SCHEDULE_HOUR ? 2 : 1);
            context.additionalInfo = `
## ⚠️ 特殊说明
现在已经接近次日（第${nextDay}天），你和村长的这次对话相当于在安排明天的计划。
请在回复的 tomorrowSchedule 字段中给出你明天的行动计划。`;
            // 取消该村民的下次系统调度
            this.scheduler.cancelNextSchedule(villager.id);
        }

        const prompt = generateVillagerPrompt(context);
        return await this.ai.chat(prompt, { temperature: 0.8, maxTokens: 500 });
    }
```

### 9.6 LLM 调用优化策略

| 策略 | 说明 | 效果 |
|-----|------|-----|
| **降级对话库** | LLM 不可用时自动切换预设对话 | 保证游戏可玩性 |
| **降级默认计划** | 调度 LLM 失败时使用预设行动序列 | 村民不会停滞 |
| **流式输出** | 村民回复逐字显示 | 掩盖延迟，提升体验 |
| **请求缓存** | 相似上下文复用近期响应 | 减少 API 调用 |
| **防抖控制** | 短时间内连续对话合并请求 | 降低成本 |
| **超时降级** | 8 秒超时自动切换预设 | 保证流畅度 |
| **调度串行化** | 4 个村民的调度 LLM 请求串行执行 | 避免并发超限 |

### 9.8 季节周期上下文压缩

为防止上下文无限增长，系统在**每个季节结束时**（每 3 个月 = 84 天）自动对所有 AI 模块的历史数据进行**压缩总结**。

#### 压缩触发时机

```
每季度最后一天（第 9/18/27/36 天）夜间自动触发：
├─ 1. 村民记忆压缩（每个村民单独压缩）
├─ 2. 市场历史压缩
├─ 3. 天气历史压缩
└─ 4. 村庄大事记生成
```

每个村民在一个季度内会积累大量对话和事件记录。压缩时，用 LLM 将其浓缩为一段简短的记忆摘要：

#### 市场历史压缩

```javascript
const compressMarketHistory = (marketEngine) => {
    // 压缩不需要 LLM，用纯数学统计
    const seasonSummary = {};

    for (const [itemId, history] of Object.entries(marketEngine.history)) {
        const prices = history.map(h => h.price);
        seasonSummary[itemId] = {
            avgPrice: Math.round(prices.reduce((a, b) => a + b, 0) / prices.length),
            maxPrice: Math.round(Math.max(...prices)),
            minPrice: Math.round(Math.min(...prices)),
            trend: prices[prices.length - 1] > prices[0] ? 'up' : 'down',
        };
    }

    // 只保留近 7 天详细数据，更早的用摘要替代
    for (const [itemId] of Object.entries(marketEngine.history)) {
        marketEngine.history[itemId] = marketEngine.history[itemId].slice(-168); // 7天
    }

    return seasonSummary;
};
```

```javascript
    const weatherLog = weatherSystem.seasonLog || [];
    const summary = {
        totalDays: weatherLog.length,
        distribution: {},  // { sunny: 15, rain: 8, ... }
        extremeEvents: [], // ['第23天暴雨导致农田受损', ...]
    };

    weatherLog.forEach(entry => {
        summary.distribution[entry.weather] = (summary.distribution[entry.weather] || 0) + 1;
    });

    // 清空季度日志，只保留摘要
    weatherSystem.seasonLog = [];
    weatherSystem.seasonSummary = summary;

    return summary;
};
```

### 10.2 村民维护成本

村民**没有日薪**，唯一的日常消耗是**粮食**：

| 村民数量 | 日粮食消耗 | 说明 |
|---------|----------|------|
| 1人 | 1🌾/天 | 初始阶段轻松供给 |
| 2人 | 2🌾/天 | 需稳定产粮 |
| 3人 | 3🌾/天 | 需扩建农田 |
| 4人（满员） | 4🌾/天 | 需高效农业体系 |

> **饥荒后果**：如果粮食库存不足以喂饱所有村民：
> - 粮食 = 0 时：全体村民心情每天 -1，体力恢复速度减半
> - 连续 3 天无粮：村民可能自行离开
> - 愚笨的村民可能不理解"没粮了"，还在要求吃东西

### 10.3 村民真正的"成本"

村民的管理难度才是真正的"成本"——性格带来的麻烦：

| 性格 | 带来的麻烦 | 应对策略 |
|-----|----------|---------|
| **懒惰** | 任务拖延、效率低、经常摸鱼 | 需要频繁催促或安排简单任务 |
| **愚笨** | 做错事（种错作物、浇错田） | 下达简洁明确的指令，避免复杂任务 |
| **叛逆** | 直接拒绝不合理指令 | 好好解释原因，或尊重其想法 |
| **悲观** | 经常抱怨、心情恢复慢 | 多安慰，避免连续高强度工作 |
| **体弱** | 干一会儿就累、需要频繁休息 | 安排轻量工作，保证粮食充足 |

### 10.4 AI 天气系统

#### AI 天气预测系统

AI 的职责是**分析当前村庄状况，决定未来 5 天内的特殊天气时间表**。AI 不发明新天气，只从当前季节的 3 种特殊天气中选择。

##### 核心规则（硬约束）

```
1. 只能选择当前季节对应的 3 种特殊天气
2. 每次事件持续天数由配置决定（固定，不可更改）
3. 相邻两次事件的开始日间隔 ≥ 5 天
4. 5 天内最多安排 2 次特殊天气（间隔 ≥ 3 天）
5. 也可以 0 次（AI 判断当前不需要特殊天气）
6. 季节切换时，重新预测（新季节的 3 种事件）
```

##### 天气系统核心逻辑

```javascript
// js/systems/WeatherSystem.js
class WeatherSystem {
    constructor(gameState, eventBus, aiService) {
        this.state = gameState;
        this.bus = eventBus;
        this.ai = aiService;

        this.currentSeason = 'spring';
        this.activeEvent = null;          // 当前生效的特殊天气（null = 默认天气）
        this.activeEventRemaining = 0;    // 特殊天气剩余天数
        this.schedule = [];               // AI 预测的未来事件时间表
        this.lastEventEndDay = -5;        // 上次特殊天气结束日（初始设为-5，确保第1天就能触发）
        this.predictionReason = '';       // AI 预测理由（玩家花金币可看）

        this.bus.on('newDay', (data) => this.onNewDay(data));
        this.bus.on('seasonChange', (data) => this.onSeasonChange(data));
    }

    // 获取当前天气效果（供其他系统读取）
    getCurrentEffects() {
        if (this.activeEvent) {
            return SPECIAL_WEATHER_EVENTS[this.activeEvent];
        }
        return SEASON_DEFAULT[this.currentSeason];
    }

    // 获取当前天气显示名
    getCurrentWeatherName() {
        if (this.activeEvent) {
            const e = SPECIAL_WEATHER_EVENTS[this.activeEvent];
            return `${e.icon} ${e.name}（还剩${this.activeEventRemaining}天）`;
        }
        const d = SEASON_DEFAULT[this.currentSeason];
        return `${d.icon} ${d.name}`;
    }

    // 每日更新
    onNewDay(data) {
        // 1. 特殊天气倒计时
        if (this.activeEvent) {
            this.activeEventRemaining--;
            if (this.activeEventRemaining <= 0) {
                const ended = SPECIAL_WEATHER_EVENTS[this.activeEvent];
                this.lastEventEndDay = data.day;
                this.activeEvent = null;
                this.bus.emit('weatherEventEnd', { event: ended });
                this.bus.emit('weatherChanged', { weather: SEASON_DEFAULT[this.currentSeason] });
            }
        }

        // 2. 检查今天是否有预定的特殊天气
        if (!this.activeEvent) {
            const scheduled = this.schedule.find(s => s.triggerDay === data.day);
            if (scheduled && data.day - this.lastEventEndDay >= 5) {
                this.activeEvent = scheduled.eventId;
                this.activeEventRemaining = SPECIAL_WEATHER_EVENTS[scheduled.eventId].duration;
                this.bus.emit('weatherEventStart', {
                    event: SPECIAL_WEATHER_EVENTS[scheduled.eventId],
                });
                // 弹窗通知
                this.bus.emit('weatherChanged', {
                    weather: SPECIAL_WEATHER_EVENTS[scheduled.eventId],
                    isSpecial: true,
                });
            }
        }
    }

    // 季节切换时，请求 AI 重新预测
    async onSeasonChange(data) {
        this.currentSeason = data.season;
        this.schedule = [];
        await this.requestAIPrediction();
    }

    // 每 5 天或季节切换时，请求 AI 预测
    async requestAIPrediction() {
        const prompt = generateWeatherPredictionPrompt(this.buildContext());
        const result = await this.ai.chat(prompt, { temperature: 0.6, maxTokens: 400 });

        if (result && result.predictions) {
            // 验证硬约束
            this.schedule = this.validatePredictions(result.predictions);
            this.predictionReason = result.overallReason || '';
        } else {
            // AI 失败时降级：随机安排 1 次
            this.schedule = this.fallbackPrediction();
            this.predictionReason = '（系统自动生成）';
        }

        this.bus.emit('forecastReady', {
            schedule: this.schedule,
            season: this.currentSeason,
        });
    }

    // 验证 AI 输出是否符合硬约束
    validatePredictions(predictions) {
        const valid = [];
        const seasonEvents = Object.values(SPECIAL_WEATHER_EVENTS)
            .filter(e => e.season === this.currentSeason)
            .map(e => e.id);

        let lastDay = this.lastEventEndDay;
        for (const p of predictions.slice(0, 2)) {  // 最多 2 个
            if (!seasonEvents.includes(p.eventId)) continue;   // 必须是本季事件
            if (p.triggerDay < 1 || p.triggerDay > 5) continue; // 必须在 5 天内
            if (p.triggerDay - lastDay < 3) continue;           // 间隔 ≥ 3 天
            valid.push(p);
            lastDay = p.triggerDay + SPECIAL_WEATHER_EVENTS[p.eventId].duration;
        }
        return valid;
    }

    // 降级方案：随机安排
    fallbackPrediction() {
        const seasonEvents = Object.values(SPECIAL_WEATHER_EVENTS)
            .filter(e => e.season === this.currentSeason);
        const pick = seasonEvents[Math.floor(Math.random() * seasonEvents.length)];
        return [{ eventId: pick.id, triggerDay: 5 + Math.floor(Math.random() * 5), reason: '随机' }];
    }
}
```

## 总结

**治村物语**重新定义了玩家与游戏世界的交互方式：

1. **不再亲力亲为** — 你是村长，通过对话指挥，而非亲手劳作
2. **每个村民都有灵魂** — AI 驱动的性格让管理充满变数和乐趣
3. **招人如抽卡** — 花 50 金币盲抽一个村民，可能是天选打工人，也可能是摸鱼大师
4. **对话就是游戏** — 催促懒惰村民干活、纠正愚笨村民的错误、说服叛逆村民服从，语言即权力
5. **指令≠结果** — 你说种萝卜，愚笨的村民可能给你种成土豆，这就是管理的乐趣
6. **村民有自己的计划** — 每隔一天 AI 自动为村民规划全天行动，懒惰的安排大量摸鱼，勤劳的排得满满当当；你随时可以对话干预修改计划
7. **看天吃饭** — AI 天气系统带来真实的不确定性，暴雨毁田、干旱缺水，天气预报帮你提前决策
8. **市场如股海** — 100 个虚拟交易者驱动的实时市场，AI 分析师每日简报，买卖时机全凭判断
8. **用人即风险** — 让聪明村民帮你炒市场赚差价，还是让笨蛋村民暴雨天冒雨出门？这就是管理
9. **三位 AI 顾问** — 天气预报员、市场分析师、个性化村民，三个 AI 模块协同驱动游戏世界
10. **精而非多** — 4 名村民的深度互动，胜过 100 个没有个性的 NPC
11. **面板即世界** — 清晰的数据面板让你专注于决策，而非操作
